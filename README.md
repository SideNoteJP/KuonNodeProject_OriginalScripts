#  **Kuon Node Project – README改良案（ドラフト）**

##  Kuon Node Projectとは  
**Kuon Node Project（KNP）は、誰でも自由に使える“オープンソースのヴィジュアルノベル台本”です。**  
舞台は **架空の1990年代日本**。  
湿度のある街、古い建物、曖昧な科学技術、静かな生活の気配。  
そんな“ノスタルジー・サイエンスフィクション”の世界を目指しています。

このレポジトリは、  
**あなたがこの世界で遊ぶための“世界観辞典”と“台本の原型”** をまとめたものです。

---

##  読者にどう遊んでほしいか  
KNPは「読むだけの資料」ではありません。  
**あなた自身がこの世界を歩き、書き、描き、改造できる“遊べる世界観”です。**

### あなたができること  
- **ショートショートを書く**  
  - 世界観辞典を参照して、好きな人物・街・建物を使って短編を作れます  
- **イラストを描く／AIで生成する**  
  - この世界の住人や建築物を自由に描いてOK  
- **設定を追加する**  
  - Wikiやテキストを編集して、世界観を拡張できます  
- **自分のゲーム・小説・動画の素材に使う**  
  - CC BY 4.0なので、商用利用も改変も自由  
- **Discordで議論・相談・共同制作ができる**  
  - 新しい視点を持つ人とつながれます  

KNPは「読む世界」ではなく、  
**“参加して遊ぶ世界”** として設計しています。

##  ファイル構成

### **基本台本ファイル**

### **README.md**  
現在閲覧いただいているファイル。

### **01_世界観の常識.txt**  
「Kuon Node Project」の世界において、どのような技術・企業・国家が存在するかを記述したファイル。

### **02_百科事典.txt**  
「もし辞書が存在したらどのような内容か」を示すファイル。  
その世界に住む一般の人が参照する「常識」をまとめています。

### **03_会話シーン例.txt**  
学校・職場・家庭など、日常風景で登場人物が交わす会話例。  
このサンプルが豊富だと **AI生成の精度・質感が大幅に向上**します。

### **04_登場人物像.txt**  
登場人物の属性、所属組織、思考回路などを定義するファイル。  
TRPGのように「今日は別キャラで遊ぶ」場合は、AI生成の初期プロンプトで変更できます。

### **05_条件定義.txt**  
AI生成において「AIは何を目指してテキストを生成すべきか」を定義するファイル。  
例：  
> 学校を舞台とした複数ヒロインとの恋愛劇。主人公は一人のヒロインに告白し、結ばれて卒業する。

### **06_構成指示.txt**  
AI生成において、  
- 無限に生成するシナリオとして楽しむか  
- 読後感のよい終わりのあるシナリオとしてまとめるか  
などを記述するファイル。

### ** AI用語集システム（新機能）**

#### **GLOSSARY.md**  **新規**
全ファイルから抽出した181用語を体系化した横断的用語集。GitHub Wiki対応済み。

#### **LOCAL_PROCESSING_GUIDE.md**  **新規**  
ローカルLLMを使った用語分類の詳細手順書。コスト$0で高品質な用語管理が可能。

#### **GITHUB_ACTIONS_SPEC.md**  **新規**
段階的自動化システムの設計書。手動→セミオート→フルオートへの移行パス。

#### **IMPLEMENTATION_SUMMARY.md**  **新規**
5分でセットアップ可能な導入ガイド。今すぐ使える機能の説明。

#### **AI処理システム（.github/workflows/）**  **新規**
GitHub Actionsによる処理状況の自動監視・レポート生成。

ノベルゲームにおける王道のゴールは  
**「一人のヒロインを選び取り、ハッピーエンドを迎える」**  
ですが、本プロジェクトでは **遊び方の自由度** を最大化するために、

- **日常レイヤ**  
- **非日常レイヤ**

を切り替えられる構造を採用しています。

平穏な日常が続くように見えても、  
ユーザーの行動や蓄積した文脈によって別レイヤへ移行し、  
**“世界の本質”が露呈する余地** を常に保持します。

これは特定作品の模倣ではなく、

- 「日常が続く保証」  
- 「急変できる余地」

の両立を目指した自由度設計です。

この構造は **「構成指示」「条件定義」** によって自由に改編できます。

---

##  レイヤ自由度の設計意図

- ユーザーが **どのレイヤで遊びたいか** を選べる  
- 選んだ後でも **別レイヤへ移行できる余地** を残す  

この2点が **Kuon Node Project の自由度設計の中心**です。
