### **02_百科事典.txt**  
「もし辞書が存在したらどのような内容か」を示すファイル。

学校・職場・家庭など、日常風景で登場人物が交わす会話例。  
このサンプルが豊富だと **AI生成の精度・質感が大幅に向上**します。

### **04_登場人物像.txt**  
登場人物の属性、所属組織、思考回路などを定義するファイル。  
TRPGのように「今日は別キャラで遊ぶ」場合は、AI生成の初期プロンプトで変更できます。

### **05_条件定義.txt**  
AI生成において「AIは何を目指してテキストを生成すべきか」を定義するファイル。  
例：  
> 学校を舞台とした複数ヒロインとの恋愛劇。主人公は一人のヒロインに告白し、結ばれて卒業する。

### **06_構成指示.txt**  
AI生成において、  
- 無限に生成するシナリオとして楽しむか  
- 読後感のよい終わりのあるシナリオとしてまとめるか  
などを記述するファイル。

ノベルゲームにおける王道のゴールは  
**「一人のヒロインを選び取り、ハッピーエンドを迎える」**  
ですが、本プロジェクトでは **遊び方の自由度** を最大化するために、

- **日常レイヤ**  
- **非日常レイヤ**

を切り替えられる構造を採用しています。

平穏な日常が続くように見えても、  
ユーザーの行動や蓄積した文脈によって別レイヤへ移行し、  
**“世界の本質”が露呈する余地** を常に保持します。

これは特定作品の模倣ではなく、

- 「日常が続く保証」  
- 「急変できる余地」

の両立を目指した自由度設計です。

この構造は **「構成指示」「条件定義」** によって自由に改編できます。

---

## 🔧 レイヤ自由度の設計意図

- ユーザーが **どのレイヤで遊びたいか** を選べる  
- 選んだ後でも **別レイヤへ移行できる余地** を残す  

この2点が **Kuon Node Project の自由度設計の中心**です。
